# 2026-07-06 — 性能优化

## 完成工作
对 OES 实验室网站进行了系统性性能优化：

### 1. 修复 init 竞态条件 bug（关键修复）
- 原来 index.html 内联脚本在 loadData() 完成前就调用渲染函数，导致空数据闪现 + 数据加载后二次渲染
- 合并了 index.html 内联脚本中的所有逻辑到 app.js，统一为单一异步 init() 入口
- 移除重复的 PAGES、navigate、showPage、toggleMobile 等函数定义
- 修复 filterMember/filterNews 使用废弃全局 event 的问题，改为接收参数

### 2. CSS 外置化
- 从 index.html 抽取 594 行 / 32KB 内联 CSS 到独立 styles.css
- HTML 从 ~32KB 减小到 25KB，且 CSS 可被浏览器独立缓存（1年 max-age）

### 3. 合并 JSON 请求
- 创建 data/data.json 合并所有 5 个数据文件
- app.js 改为优先请求单文件，失败时回退到分文件请求
- 5 次 HTTP round-trip 减少到 1 次

### 4. 服务端性能
- 添加 ETag 支持条件请求（304 Not Modified）
- 添加 Gzip 压缩（HTML/CSS/JS/JSON/SVG）
- 添加 Cache-Control 头（静态资源 1 年，HTML 不缓存）
- 关闭 CORS 通配符，限制为 localhost:8080
- 服务端保存后自动重新生成 data/data.json
- 启动时自动生成 data/data.json

### 5. SEO + 懒加载
- 添加 meta description、keywords
- 添加 Open Graph 标签
- 添加 canonical URL
- Google Fonts 改为异步加载（preload + onload 模式，带 noscript 回退）
- 所有 JS 渲染的 img 标签添加 loading="lazy" + decoding="async"
- 添加 esc() 函数进行基本 XSS 防护
- 轮播图添加 ARIA role 和 aria-label

### 6. 修复数据加载问题 (21:40)
- **问题**: 改为优先加载 data.json 合并文件后，用户反馈网站无数据
- **原因**: 合并文件加载策略不够稳健，作为主要数据源不够可靠
- **修复**: 改为三层加载策略：
  1. 主策略：并行加载 5 个独立 JSON 文件（原方案，已验证可靠）
  2. 回退：任一个体文件失败时尝试合并 data.json
  3. 保底：全部失败后最终尝试合并文件
- 确保不影响其他优化（CSS 外置、Gzip、ETag、SEO 等全部保留）

## 关键文件变更
- `index.html`: 内联 CSS 外置，移除重复 init，添加 SEO 标签
- `styles.css`: 新建（594 行）
- `src/js/app.js`: 合并内联逻辑，修复 bug，添加 esc()，使用单文件数据加载
- `start-server.js`: 添加 ETag、Gzip、Cache-Control、data.json 自动生成
- `data/data.json`: 新建（合并文件）
